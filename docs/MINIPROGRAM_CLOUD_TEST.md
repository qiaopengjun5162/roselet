# 小程序云测验收流程

> 快速开始：https://developers.weixin.qq.com/miniprogram/dev/devtools/minitest/fast_start.html
> AIMonkey：https://developers.weixin.qq.com/miniprogram/dev/devtools/minitest/ai_monkey.html
> AI 自定义测试：https://developers.weixin.qq.com/miniprogram/dev/devtools/minitest/ai_plan.html

## 适用范围

微信官方云测在使用前需要满足：

- 必须有体验版或线上版小程序，不支持测试普通开发版。
- 当前用户需要是该小程序的开发者或管理员。
- 只支持小程序，不支持小游戏。

## 打开方式

- 推荐方式：在微信开发者工具中安装“云测”插件。这样可以同步开发者工具里的小程序和用户信息。
- 备用方式：直接打开云测 Web 地址并微信扫码登录。

## 提测前 Roselet 检查清单

1. 执行 `just wasm-mini`，确保小程序 WASM 构建和 WXWebAssembly patch 完成。
2. 执行 `cd apps/miniprogram && pnpm typecheck`。
3. 执行 `cd apps/miniprogram && pnpm test:coverage`。
4. 执行 `just miniprogram-build`。
5. 在微信开发者工具导入 `apps/miniprogram/dist/weapp`。
6. 确认后端已启动：`cargo run -p roselet-backend`。
7. 上传体验版后再进入云测。

## 云测重点关注

- 登录 / 双令牌刷新是否正常。
- 种花流程是否能提交。
- 私密模式开关和锁标识是否正常。
- 公共花圃不展示私密玫瑰。
- 我的花圃能展示自己的私密玫瑰。
- WASM 花瓣、颜色、表单校验是否正常。
- 报告中的 JsError、黑白屏、截图和性能达标率。

## AIMonkey

AIMonkey 是微信云测里的 AI 自动化测试模式，适合作为 Roselet 小程序的探索式回归测试。

### 官方要点

- AIMonkey 依托大模型能力驱动测试，比传统智能化 Monkey 更接近人类感知。
- AIMonkey 执行完成后会自动生成本次操作的 Minium 代码。
- 生成的 Minium 代码可以下载，再上传为 Minium 用例，用于回放本次 Monkey 过程。
- 鸿蒙操作系统只支持 AIMonkey 模式；即使页面选择智能化 Monkey，云测也会自动切到 AIMonkey。
- AIMonkey 不支持原智能化 Monkey 的拓展能力，例如不支持配置前置步骤。

### Roselet 使用建议

1. 提交云测任务时，在 Monkey 测试类型的测试计划中选择 `AIMonkey`。
2. 优先覆盖不需要复杂前置状态的路径：
   - 首屏 / 导航栏
   - 花圃浏览
   - 颜色筛选
   - 关于页 / 反馈页
   - 登录入口和错误态
3. 对需要固定账号状态的路径，优先用“测试账号”方案或 Minium 自定义用例，不依赖 AIMonkey 前置步骤。
4. AIMonkey 跑完后检查“自动生成用例”Tab；如果发现高价值路径，把生成的 Minium 代码沉淀为可回放用例。
5. 如果报告发现黑白屏、JsError、WASM 加载失败、页面卡死，先在本地微信开发者工具复现，再补 Jest / Minium 回归测试。

### Roselet 重点风险

- WASM 初始化失败后是否有可见错误或降级。
- 登录态过期后是否能静默刷新，刷新失败是否回登录页。
- 私密模式是否会被 AIMonkey 随机操作误提交。
- 花圃 WebSocket / 本地缓存恢复是否导致重复玫瑰或过期 UI。
- 表单输入为空、超长、切换颜色、多次点击提交时是否稳定。

## AI 自定义测试

AI 自定义测试适合把明确的 Roselet 用户路径写成自然语言用例，再由云测用 Minium 驱动执行。它比 AIMonkey 更适合“我要验证这条路径”的定向回归。

### 官方要点

- AI 自定义测试目前处于内测中。
- 当前 AI 操作支持点击、简单输入、滑动、`function_call`、大模型断言等能力。
- 新建 AI 用例时，需要填写用例名称、优先级和用例描述。
- 用例描述就是本次 AI 探索过程的自然语言描述，应尽量准确、详细。
- 描述可以从人类视觉角度编写；云测会把当前屏幕截图给大模型，可以假设小程序已经打开，只描述后续步骤。
- AI 测试计划会按勾选顺序执行用例，并采用 DDT 模式依次执行。
- 提交任务时选择 `Minium` 测试类型，再选择带 AI 标识的测试计划。
- 测试成功表示 AI 按描述自认为执行完成，仍需人工结合报告截图和生成的 Minium 代码判断。
- 测试失败常见原因包括任务描述要求失败、AI 连续探索失败、AI 探索超时。

### 描述用例 Tips

- 滑动时写清楚上下滑动还是左右滑动。
- 需要到底部时，直接描述“滑动到页面底部”。
- 需要在滚动中找文字时，写“向下滑动，找到 xxx”；目标文字必须确实存在，因为云测会用 OCR 匹配。
- 断言用 `llm_assert("...")`；断言失败会终止当前 Case。
- 成功执行后，下载生成的 Minium 代码，优先把稳定路径转成可复跑用例。

### 特殊能力

- `minium_func_call`：可以在自然语言描述中调用自定义函数；函数必须先上传到云测。
- 自定义函数第一个参数必须是 `mini`，云测会传入 `minitest.MiniTest` 实例。
- 自定义函数参数尽量使用简单数值或字符串，避免复杂数组/字典。
- 文件名、路径、函数名尽量使用英文，不带特殊符号，避免 import 失败。
- 需要在报告里看到函数日志时，使用 `mini.user_logger.info(...)` 或其他 `user_logger` 等级。
- `skills`：适合注册一组技能，让大模型根据当前场景自行判断是否调用。

### Roselet 推荐 AI 用例

1. 登录与资料页：
   - 从首页进入登录页。
   - 输入一个测试昵称完成登录。
   - 进入个人资料页。
   - `llm_assert("页面展示用户昵称和玫瑰统计")`。
2. 种花基础流程：
   - 进入种花页。
   - 选择红玫瑰。
   - 输入一段感恩文本。
   - 提交种花。
   - `llm_assert("出现种植成功反馈或进入成功状态")`。
3. 花圃筛选：
   - 进入花圃页。
   - 点击红玫瑰筛选。
   - 点击白玫瑰筛选。
   - 点击全部。
   - `llm_assert("筛选按钮可切换，页面没有黑白屏或错误提示")`。
4. 私密模式冒烟：
   - 登录后进入种花页。
   - 打开私密模式。
   - 输入文本并提交。
   - 进入我的花圃。
   - `llm_assert("我的花圃中可以看到刚提交的私密玫瑰或锁标识")`。

### Roselet 使用边界

- 需要固定登录态、固定后端数据或固定昵称时，优先用测试账号或 `minium_func_call` 准备状态。
- AI 自定义测试不是接口测试；如果失败来自后端状态不稳定，要先补 Rust 后端集成测试。
- AI 生成的 Minium 代码需要代码审查后再纳入长期回归，不直接当作可信测试源。
- 对于私密模式、登录刷新、WASM 校验等关键逻辑，最终仍要落到 Rust / Jest / Minium 的确定性测试。

## 经验规则

云测不是本地 Jest 的替代品。它用于覆盖真机环境、兼容性、黑白屏、性能和弱网络等问题；本地单元测试仍然是提交前门禁。

AIMonkey 适合发现未知路径问题，但不能替代确定性回归测试。发现稳定复现的问题后，要沉淀成 Rust 单元测试、Jest 测试或 Minium 用例。

AI 自定义测试适合把明确业务路径快速写成自然语言用例；当路径稳定后，应下载 Minium 代码并整理成长期回归用例。
